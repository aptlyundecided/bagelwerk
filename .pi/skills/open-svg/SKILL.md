# Open SVG

Bypass the headless terminal sandbox to pop open an SVG file directly on the operator's host operating system, using strict file validation.

## 1. Do not use generic powershell commands
Because the agent runs in an isolated, headless sandbox on Windows, standard commands like `Start-Process <file>.svg` or `explorer.exe` will often execute invisibly in the background and will NOT appear on the operator's screen due to Windows session isolation.
The `openSvg.ts` script explicitly bypasses this by using `cmd.exe /c start "" "<file>"` which reliably pierces the background session to trigger the Windows Desktop Shell. Always use this skill instead of trying to open files manually.

## 2. Request Unsandboxed Permission

Whenever instructed to "open", "show", or "view" an SVG file visually for the user:
1. Ensure the file actually exists and ends with `.svg`.
2. Use the `ask_permission` tool:
   - `Action`: `unsandboxed`
   - `Target`: `npm run open-svg`
   - `Reason`: `To pop open the SVG file visibly on the host OS`

## 3. Execute the CLI Command

Once permission is granted, run the following command using the `run_command` tool:

```bash
npm run open-svg <absolute-path-to-svg>
```

The underlying script (`openSvg.ts`) runs on the host system. It will strictly validate that the file is safe to open (checking the extension and asserting it contains an `<svg` tag) before using Node's native functionality to launch the default viewer.

## Anti-patterns
- Do not attempt to read the file contents to the chat using `view_file` unless explicitly asked to view the *source code*. If they ask to "open" the SVG, they want to see the graphic.
- Do not forget to request `unsandboxed` permission first; otherwise, the host script will be blocked from launching a graphical window on the operator's display.
