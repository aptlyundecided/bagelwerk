# 0.4.0 - 2026-06-01

## Added

- Generic Flow Runner entrypoints for external Flow libraries (`npm run flow:runner`) and PR Magic (`npm run flow:pr-magic`).
- Generic Flow Runner Ink progress view for Flow/Node execution and Node-internal progress events.
- Default validation-informed retry/healing loop for agent skill-backed Nodes.
- npm package renamed to **`flow-smithy`** (display name **Flowsmithy**).
- Added the `version-release` skill for SemVer bumps and changelog/release-note prep.
- Added a `docs/` orientation hub for future developers and agents.
- Added `scripts/active-surface-test-run/about.md` to document the active test selector.

## Changed

- `pr-magic` is now the only active built-in Flow package under `src/core/built-ins/`.
- PR Magic runs through Generic Flow Runner profiles by default; the old direct PR Magic v1 path and temporary migration facade were removed.
- PR Magic behavior duplicate checks now treat repeated `title + afterBehavior` matches as advisory warnings while repeated surviving `findingId`s remain blocking validation errors.
- Changelog material now lives under `changelog/` with separate current notes and release files instead of a single root `CHANGELOG.md`.
- Active-surface test runner moved from `scripts/active-surface-test-run.mjs` to `scripts/active-surface-test-run/run.mjs`.

## Removed

- Removed the active onboarding, Flow Doctor, GitHub PR setup, guided Flow builder, and submit-current-work built-in packages from `src/core/built-ins/`.
- Removed deleted-surface shortcuts such as `npm run intro`, `npm run flow:smithy`, `npm run flow-workbench:built-ins`, and `npm run flow:work-submission`.
- Removed legacy PR Magic aliases `npm run flow:pr-magic:v2` and `npm run flow:pr-magic:runner`; use `npm run flow:pr-magic`.
- Removed stale `.pi/agents/` role prompts from the retired Taskplane-style orchestration setup.
- Removed adoption docs under `docs/adoption/`; early-adopter guidance now starts from the root `README.md`, `CONTEXT.md`, and local package READMEs.
